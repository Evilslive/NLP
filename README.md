# NLP

## Mandarin Word Segmentation

sf版本- &gt; 不預定資料庫, 直接透過文本所提供的訊息進行處理分析

### Base On Frequency Theory

經常組合出現的字視為詞, 頻率較高者先切出

* ex: 今天是下雨天, \|今天\|是\|下雨天\|, 「今天」與「是」的切割較沒有爭議, 但「下雨天」三個字可以分成下雨、雨天、下雨天三個詞, 如果文本中「下雨天」出現的次數跟其他兩者相同、甚至較高的情況時, 則處理結果會切出「下雨天」。
* 另一種情況, 若「雨」這個單字出現的較多次, 則下雨天三個字會分別拆開為單字詞, 但若文本其他地方出現下雨、或雨天, 則還是會切出雙字詞, 完全依照文句脈絡決定。

  **Purposes**

* 不使用Jieba、或預設參照資料庫，檢驗文本斷詞的中文語料分析系統。
* 僅切分出1-4字詞\(單字詞、雙字詞、三字詞、四字詞\)，多數五、六字詞以上均可再切短。
* 以文章的用字遣詞計算共現性，使得斷詞不受世代效果的影響，但切出的字詞受寫作習慣影響\(作者風格?\)。

  **Issues**

* 數量詞: 如「七輛囚車、三輛囚車」，都切為數字+輛囚車。  

  反對purpose1 &gt; 部分單獨使用的字，如「輛」，應該先篩選出來。  

  支持purpose1 &gt; 在文本量大之後\(多大?十萬以上?\)，有機會排除問題，如出現「囚車」的次數比「輛囚車」多。  

* 介詞:的、是、把，透過探討可能的組合詞，先行切出。\(如Issue1所述\)
* 為了解決前兩點的問題，新增自定義字典，將預切的字詞放入。
