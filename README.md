# NLP

### Chinese Word Segmentation
self- > 不預定資料庫, 直接透過文本所提供的訊息進行處理分析

#### Base On Frequency Theory
經常組合出現的字視為詞, 頻率較高者先切出
ex: 今天是下雨天, |今天|是|下雨天|, 「今天」與「是」的切割較沒有爭議, 但「下雨天」三個字可以分成下雨、雨天、下雨天三個詞, 如果文本中「下雨天」出現的次數跟其他兩者高、甚至較高的情況時, 則處理結果會切出「下雨天」。
另一種情況, 若「雨」這個單字出現的較多次, 則下雨天三個字會分別拆開為單字詞, 但若文本其他地方出現下雨、或雨天, 則還是會切出雙字詞, 完全依照文句脈絡決定。

#### Purposes
1. 試著不用Jieba，做出更好的斷詞等中文語料處理分析系統。
2. 僅切分出1-4字詞(單字詞、雙字詞、三字詞、四字詞)，五、六字詞以上多數可以再切短一點。
3. 以文章的用字遣詞計算共現性，使得斷詞不受世代效果的影響，但切出的字詞受寫作習慣影響(作者風格?)。

#### Issues
1. 數量詞: 如「七輛囚車、三輛囚車」，都切為數字+輛囚車。或許中文的數字「一、二、三...」，要獨立拉出來。
2. 介詞:的、是、把，透過探討可能的組合詞，先行切出。
3. 為了解決a&b的問題，日後增建"rf"版本，新增自定義字典，將預切的字詞放入(那就又回到Jieba的模式了)。
4. 處理速度的問題，目前的處理方式會形成檔案四倍大的矩陣，再計次、整合上會較慢
